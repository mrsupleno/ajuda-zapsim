# API — Oportunidades

## Criar Oportunidade

```
POST /v2/api/external/{ApiID}/createOpportunity
```

**Body JSON:**

```json
{
  "contactId": 123,
  "title": "Proposta Plano Premium",
  "value": 500.00,
  "pipelineId": 1,
  "stageId": 2,
  "userId": 5
}
```

## Parâmetros

| Parâmetro | Descrição |
|-----------|-----------|
| `contactId` | ID do contato relacionado |
| `title` | Título da oportunidade |
| `value` | Valor estimado da venda |
| `pipelineId` | ID do pipeline (funil) |
| `stageId` | ID do estágio (etapa do funil) |
| `userId` | ID do usuário responsável |

## Atualizar Oportunidade

```
POST /v2/api/external/{ApiID}/updateOpportunity
```

**Body JSON:**

```json
{
  "opportunityId": 45,
  "stageId": 3,
  "value": 750.00,
  "userId": 8
}
```

## Deletar Oportunidade

```
POST /v2/api/external/{ApiID}/deleteOpportunity
```

**Body JSON:**

```json
{
  "opportunityId": 45
}
```

## Kanban e Pipeline

O sistema de oportunidades funciona com um pipeline de vendas no formato Kanban:

1. **Pipeline:** Funil de vendas completo
2. **Stages:** Etapas do funil (ex: Qualificação, Proposta, Negociação, Fechado)
3. **Cards:** Oportunidades individuais

### Estágios Típicos

| Estágio | Descrição |
|---------|-----------|
| 1 | Novo Lead |
| 2 | Qualificação |
| 3 | Proposta Enviada |
| 4 | Negociação |
| 5 | Fechado Ganho |
| 6 | Fechado Perdido |
